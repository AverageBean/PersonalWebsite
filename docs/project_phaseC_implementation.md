## Phase C-1 — DONE (2026-04-29)

Implemented in `tools/convert-stl-to-step-parametric-with-freecad.py`:

- `fit_ellipse_2d(points)` — direct algebraic ellipse fit (Fitzgibbon, Pilu,
  Fisher 1999) via constrained generalized eigenvalue problem.
- `ellipse_distances(points, fit)` — radial-scaling perpendicular-distance
  approximation for inlier classification.
- `detect_elliptic_cylinders(face_centers, face_normals, used, total,
  mesh_triangles)` — RANSAC-style iterative detection on lateral
  (|n·Z| < 0.30) unclaimed faces. Validates axis ratio b/a < 0.95 to avoid
  reclaiming circular cylinders. Z extents come from inlier triangle
  vertices (not centroids — centroid Z range is a tighter inner bound).
- `build_elliptic_cylinder_solid(elliptic_cyls, height_extension=0.0)` —
  CSG path: builds `Part.Ellipse` curves, wires them, extrudes along the
  detected axis. Native STEP export emits `SURFACE_OF_LINEAR_EXTRUSION` on
  an `ELLIPSE` curve.
- Routing in `build_parametric_solid` — elliptic-cylinder path takes
  precedence over the cylindrical/sphere paths (Phase A may have
  spuriously fitted circular cylinders to subsets of an elliptic wall).
- Synthetic test STL `TestDocs/AIgen_EllipticCylinder.stl` (a=10, b=6, h=30)
  generated by `tools/generate-elliptic-cylinder-test.py`.
- Regression entry in `tools/test-parametric-step.py` — vol ratio
  0.98–1.02, max mean dev 0.05 mm, requires `ELLIPSE` and
  `SURFACE_OF_LINEAR_EXTRUSION` STEP entities.

Two implementation gotchas captured here so the C-2 work doesn't re-walk
them:

1. **Fitzgibbon eigenvalue indexing.** `np.linalg.eig` returns eigenvectors
   as columns: the constraint check `4AC - B² > 0` must use
   `eigvec[0, :] * eigvec[2, :] - eigvec[1, :]**2`, not the row form. Got
   silent "fit degenerate" failures with the row form because none of the
   rows passed.
2. **Canonical-form sign.** The semi-axis formulas have a sign flip vs the
   raw `B² − 4AC` expression: `a² = -num / (denom · ((A+C) + s))`. Without
   the leading `-`, both `a²` and `b²` come out negative for valid ellipse
   fits and the function returns None.

## Phase C-2 — DONE (2026-05-05)

Implemented in `tools/convert-stl-to-step-parametric-with-freecad.py`:

- `find_revolution_axis(face_centers, face_normals, used, total)` —
  scores 4 candidate axes (X, Y, Z, PCA principal). Per-axis score
  combines inlier-fraction × theta-coverage / (1 + r-MAD/r-scale).
  Returns origin + axis + per-z-bucket profile + inlier mask, or None.
- `_score_revolution_axis(...)` — bucket-based projection into
  cylindrical (z, r, θ); per-bucket median r and MAD; theta-coverage
  uniformity check.
- `extract_profile_zr(profile_buckets, z_extent)` — clean (z, r) array,
  median-binned per z-bucket. Critical: extends profile to true z-extent
  by anchoring nearest-bucket r at the bounds (without this the resulting
  solid under-fills by ~1 bucket-width on each end).
- `fit_revolution_profile(zr_pts)` — `Part.BSplineCurve.approximate`
  with degree ≤ 5, tolerance 0.05 mm. Reject gates:
    - r-stdev < 0.05 mm → cylinder, defer to Phase A
    - max r-jump / r-range > 50% → annular ring, defer
    - sign-changes < 1 → monotonic profile (cylinder/cone), defer
    - residual > 4 × tolerance → silent bad fit, reject
- `build_revolution_solid(rev, profile_fit)` — profile wire (B-spline +
  axis-closing edges, skipping degenerate edges where r ≈ 0 — pole edge
  case) → `Part.Face` → revolve 360° around local Z. `FreeCAD.Placement`
  rotates local-Z onto world axis and translates to world origin.
- Pipeline integration: detection runs after sphere/elliptic, on the
  `used_after_planes` mask (sees full lateral face set). Routing in
  `build_parametric_solid` gives revolution **first priority** ahead of
  box, elliptic, cylindrical paths.
- `--debug-c2` CLI flag — verbose per-axis scoring + per-stage reject
  reasons.
- `tools/debug-revolution-fit.py` — standalone harness (trimesh + numpy
  + matplotlib only, no FreeCAD). Validates axis selection on any STL
  before the full converter runs.
- Synthetic test `TestDocs/AIgen_RevolvedOgive.stl` from
  `tools/generate-revolved-profile-test.py` (vase profile through 6 (r, z)
  control points, B-spline interpolated, revolved 360° around Z, ASCII
  STL with `solid AIgen_RevolvedOgive` header).
- `TestDocs/README_AI_GENERATED.md` — sidecar registering AIgen_ assets.
  Existing `EllipticCylinder.stl` renamed to `AIgen_EllipticCylinder.stl`
  for naming consistency.
- Regression entry tightened to vol ratio 0.98–1.02, mean dev ≤ 0.20 mm,
  required entities `SURFACE_OF_REVOLUTION`, `B_SPLINE_CURVE_WITH_KNOTS`.

Five implementation gotchas captured in
`LearningLog/2026-05-05_PhaseC2_Revolutions.md`:

1. Post-claims mask sees nothing — Phase A cylinder RANSAC partially
   fits each height slice of a vase as a "cylinder". Solution: detect
   on `used_after_planes`.
2. Annular rings produce per-bucket median noise that scores as a
   revolution. Solution: max-r-jump / r-range gate.
3. `Part.export([raw_shape], path)` writes empty STEP — needs Document
   features wrapped via `addObject("Part::Feature", ...).Shape = sol`.
4. Profile (z, r) sampled at bucket centres under-fills the solid.
   Solution: `extract_profile_zr` extends to true z_extent.
5. Unicode arrows in console prints crash Windows cp1252.

## Why C-3 stays deferred

The spec lists C-3 (B-spline fallback) as the catchall for arbitrary curved
surfaces. Two reasons not to jump there next:

1. **C-3 needs B-spline surface fitting.** FreeCAD's Part.BSplineSurface
   can be constructed from a control-point grid, but generating that grid
   from arbitrary face centres requires either (a) parameterising the surface
   onto a 2D domain (hard for general topology) or (b) using a library like
   `geomdl`. Both are a real day or two of work.
2. **C-3 trim curves are non-trivial.** A B-spline patch needs to be trimmed
   to the actual silhouette of the unclaimed face cluster. `BRepAlgoAPI_Section`
   handles this but produces complex topology that can fail boolean ops.

C-2 should land first — same projection machinery as C-1 plus a known
1D-fit problem (revolved profile). C-3 is the bigger bet.
